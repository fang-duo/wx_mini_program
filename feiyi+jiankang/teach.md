# 遗韵养生小程序开发、上线与运营说明

## 1. 先明确这个项目的目标

基于当前项目，建议把目标定为下面这个模式：

- 用户端：扫码即可使用，不强制注册登录。
- 用户数据：打卡记录、AI 历史、收藏等优先保存在用户自己手机本地。
- 运营端：首页宣传图、宣传视频、活动内容不写死在代码里，而是放到微信云开发里，由运营在微信云开发控制台手动增删改。
- AI 调用：前端不直接请求 DeepSeek，而是由小程序调用云函数，云函数再去请求 DeepSeek。

这是一种适合当前项目的“轻量化 + 可运营”的方案。

## 2. 当前项目已经做到的部分

当前项目已经有这些基础：

- 首页、健康打卡、AI 问答、个人中心四个主页面。
- 本地打卡记录和目标设置。
- AI 问答页面界面。
- AI 问答历史、本地收藏、批量删除。
- 微信云开发初始化代码。
- 云函数 `aiChat` 的基本雏形。

对应代码位置：

- 小程序云环境初始化：`app.js`
- AI 页面：`pages/ai/ai.js`
- 首页静态内容：`pages/index/index.js`
- AI 云函数：`cloudfunctions/aiChat/index.js`

## 3. 现在 AI 无法调用的根本原因

从目前项目状态看，AI 调不起来，至少有下面几个问题。

### 3.1 云环境没有配置成功

报错里已经明确显示：

`Environment not found`

这说明虽然 `app.js` 里写了环境 ID，但当前微信开发者工具没有成功连接到这个云开发环境，或者这个环境本身不存在/无权限。

当前代码位置：

- `app.js`

当前写法是：

```js
wx.cloud.init({
  env: 'cloud1-9glzvmt445de4677',
  traceUser: true,
})
```

这一步如果环境 ID 无效，后面的 `wx.cloud.callFunction` 一定失败。

### 3.2 云函数虽然写了，但不等于已经部署成功

`pages/ai/ai.js` 里调用的是：

```js
wx.cloud.callFunction({
  name: 'aiChat',
  data: { userInput: text }
})
```

这要求你必须已经做完下面这些事：

- 已开通微信云开发。
- 已创建云环境。
- 已在微信开发者工具里选中这个环境。
- 已上传并部署 `cloudfunctions/aiChat`。
- 已给云函数安装依赖，例如 `axios`。

只写代码但没部署，前端也会调用失败。

### 3.3 DeepSeek key 不能放在前端，也不建议明文写死在仓库里

你现在的 `cloudfunctions/aiChat/index.js` 里已经把 key 直接写死在代码里了，这个方式能临时测试，但不适合上线。

原因有两个：

- 一旦代码被看到或上传到公开仓库，key 就泄露了。
- 后续要换 key、停用 key、做多环境区分时会很麻烦。

更稳妥的做法是：

- key 只放在云函数端。
- 不放在小程序前端。
- 最好放在云函数的环境变量或私有配置里。

### 3.4 `app.json` 里把云函数页面当成普通页面注册了

当前 `app.json` 里有这一项：

```json
"cloudfunctions/aiChat/ai"
```

这不应该出现在 `pages` 数组里，因为云函数不是小程序页面。

这不是这次 AI 调用失败的直接原因，但这是一个项目结构问题，后面建议修掉。

## 4. 推荐的最终架构

如果你要实现“开发到上线到运营”的完整模式，建议按下面的结构来做。

### 4.1 用户数据

存本地：

- 健康打卡
- 体重
- 每日目标
- AI 历史
- 收藏记录

优点：

- 用户扫码即用。
- 不需要做复杂账号系统。
- 使用门槛最低。

缺点：

- 用户删除小程序或清理缓存后，数据会丢失。

这个缺点如果你可以接受，就可以继续保持轻量方案。

### 4.2 首页宣传内容

放云端：

- Banner 图片
- 宣传视频
- 活动列表
- 首页推荐文案

建议用微信云开发来存：

- 图片和视频：放云存储
- 标题、简介、排序、跳转链接：放云数据库

这样运营只需要进入微信云开发控制台，就可以改首页展示内容。

### 4.3 AI 能力

必须走云函数代理：

- 小程序前端 -> 云函数 `aiChat`
- 云函数 `aiChat` -> DeepSeek API

不能让前端直接带 key 请求 DeepSeek。

## 5. 一步一步开发

## 第一步：准备微信小程序正式环境

你需要准备：

1. 一个可用的小程序 AppID。
2. 一个已认证或可发布的小程序主体。
3. 微信开发者工具。
4. 微信云开发环境。

操作建议：

1. 打开微信开发者工具。
2. 导入当前项目。
3. 确认项目 `AppID` 不是纯测试号，而是你准备上线的小程序 AppID。
4. 点击工具栏里的“云开发”。
5. 创建一个云开发环境。
6. 记下环境 ID。

然后把 `app.js` 里的 `env` 改成你真实创建成功的环境 ID。

## 第二步：先把 AI 调用打通

这是当前最优先要解决的功能。

### 2.1 云函数目录确认

当前已有目录：

- `cloudfunctions/aiChat/index.js`

这个目录是对的。

### 2.2 安装云函数依赖

在微信开发者工具里右键 `cloudfunctions/aiChat`：

- 选择“在终端中打开”
- 执行依赖安装

常见命令：

```bash
npm install
```

### 2.3 正确部署云函数

在微信开发者工具中：

1. 右键 `cloudfunctions/aiChat`
2. 选择“上传并部署：云端安装依赖”

注意，不是只保存文件，而是要真正部署。

### 2.4 正确配置 DeepSeek

你的云函数目前请求地址思路是对的，核心是这几个点：

```js
url: 'https://api.deepseek.com/chat/completions'
model: 'deepseek-chat'
```

建议最终使用下面这个思路：

- key 只保存在云函数端
- 云函数读取私有配置
- 前端只传用户问题

也就是说，前端永远不要出现 `DeepSeek API Key`。

### 2.5 你现在报错的直接修复方向

当你看到：

`cloud.callFunction:fail Error: errCode: -501000 Environment not found`

请按这个顺序检查：

1. 微信开发者工具是否登录了正确的微信号。
2. 当前项目绑定的小程序 AppID 是否正确。
3. 是否已经开通云开发。
4. `app.js` 中的环境 ID 是否真实存在。
5. 开发者工具顶部“云开发”里是否已经切换到该环境。
6. `aiChat` 云函数是否已经上传部署成功。

只要第 4 或第 6 步没完成，AI 一定不能用。

### 3.5 运营怎么改内容

运营人员以后不需要改代码，只需要：

1. 登录微信云开发控制台。
2. 在“存储”里上传新图片和视频。
3. 在“数据库”里新增或修改对应记录。
4. 用户重新打开小程序时，首页就会加载新内容。

这就是你要的“后台改图，小程序自动跟着变”。

## 第四步：开发和联调流程

建议你以后按这个顺序开发：

1. 先完成页面静态 UI。
2. 再接本地存储逻辑。
3. 再接云数据库动态内容。
4. 最后接 AI 云函数。
5. 每做完一块，就在真机和开发者工具里各测一次。

建议优先级如下：

1. 修复 AI 调用
2. 首页改为云端动态内容
3. 修复图标和细节 UI
4. 清理无效页面配置
5. 做上线前检查

## 第五步：上线前必须完成的事

### 5.1 内容合规

你的小程序如果要上线，至少要准备：

- 小程序名称
- 小程序头像
- 服务类目
- 隐私政策
- 用户协议

如果页面里有 AI 问答，还要注意：

- 避免医疗诊断式表述
- 不要承诺治疗效果
- 更适合定位为“养生建议、文化科普、日常参考”

### 5.2 小程序后台配置

进入微信公众平台后台，逐项检查：

1. 服务器域名
2. 业务域名
3. 下载文件合法域名
4. 隐私协议配置
5. 类目和功能说明

如果你主要使用云开发，很多接口不需要自己额外买服务器，但内容和类目依然要合规。

### 5.3 上传审核

流程通常是：

1. 在微信开发者工具中点击“上传”
2. 填写版本号和备注
3. 到微信公众平台提交审核
4. 审核通过后发布

## 第六步：简单运营怎么做

你希望的是“简单运营”，那就不要一开始做复杂后台。

最轻的运营方式如下：

### 6.1 运营人员只做 3 件事

1. 上传新的首页图片
2. 上传新的宣传视频
3. 修改首页活动标题、简介、排序

### 6.2 运营入口

直接使用微信云开发控制台：

- 存储：管理图片、视频
- 数据库：管理首页配置数据

### 6.3 运营更新流程

标准流程可以固定成：

1. 设计给运营图片和视频素材
2. 运营上传到云存储
3. 运营复制云文件地址
4. 运营到数据库里修改对应内容
5. 自己扫码检查首页显示是否正常

这样就不用重新发版。

## 7. 你现在要记住的最重要结论

### 结论一

你要的模式是能做成的，而且不需要额外开发一个复杂后台。

直接用微信官方的：

- 微信公众平台后台
- 微信云开发控制台

就足够支撑“简单运营”。

### 结论二

用户数据完全可以继续保存在本地，这样可以保持轻量化。

### 结论三

首页宣传图和视频不要再写死在代码里，而要放到云开发里，这样运营才能真正接手。

### 结论四

AI 现在不能用，最直接的问题不是 DeepSeek 本身，而是你的云环境没有正确连通和部署完成。

## 8. 额外提醒

你在对话里给出了 DeepSeek 的 API Key。这个 key 不建议继续明文传播，也不要放到前端代码里。

建议你后续立即做两件事：

1. 到 DeepSeek 平台更换或重置这个 key。
2. 后续只把新 key 放进云函数私有配置中。

这样更安全，也更适合上线。

## 9. 运营同学如何用网页版云开发控制台维护内容

如果后续不希望运营同学使用微信开发者工具，可以改为使用网页版云开发控制台。

推荐做法：

1. 使用项目管理员账号登录微信云开发网页版控制台，或从微信开发者工具的云开发页面进入网页版控制台。
2. 进入你当前项目绑定的环境，例如 `cloud2-d5gq377icbed53c7e`。
3. 在网页版控制台中，运营主要只需要使用两个模块：
   - 云数据库
   - 云存储

运营维护流程建议固定为：

1. 先在云存储上传图片或视频。
2. 复制上传后生成的云文件地址。
3. 到云数据库对应集合中新增或修改记录。
4. 保存后重新打开小程序首页检查效果。
