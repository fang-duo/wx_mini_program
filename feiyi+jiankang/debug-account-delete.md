# 账号注销问题归档说明

本文档为历史调试记录归档。

## 一、当前状态

- 当前项目主流程中并未把“账号注销”作为当前上线必须功能。
- 本文档保留的目的，是提醒后续维护者：如果未来重新启用账号注销功能，需要重新验证前端、云函数、数据库删除策略和云存储头像删除逻辑。

## 二、为什么保留这份文档

曾经出现过“前端调用注销后提示失败”的问题，但当前版本的上线重点不在该功能，因此不建议在提审前临时扩展这条链路。

## 三、如果后续要继续做

建议按下面顺序重新排查：

1. 确认前端是否仍保留注销入口。
2. 确认云函数是否真实存在并已部署。
3. 确认要删除的集合范围。
4. 确认头像云存储文件删除逻辑。
5. 确认失败时能返回明确错误原因。
6. 真机验证注销后的本地缓存清理与重新登录表现。

## 四、当前建议

- 这份文档视为历史归档，不作为当前上线阻塞项。
- 如果未来重新启用账号注销，请重新编写正式设计与测试清单。
