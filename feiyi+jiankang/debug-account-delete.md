# [OPEN] account-delete

## 现象

- 前端点击“注销账号”后，弹出“注销失败，请稍后重试”。
- 控制台堆栈显示在 `profile.js` 中抛出 `Error: 账号注销失败`。
- 说明前端已成功调用到 `deleteAccount` 云函数，但拿到的返回结果中 `success` 不为 `true`，且 `error` 可能为空或未透出。

## 当前假设

1. `deleteAccount` 云函数在运行时进入了 `catch`，但返回的 `error.message` 为空，导致前端只能拿到通用错误。
2. 云函数里对 `_openid` / `openid` 的组合查询在某个集合上触发了运行时异常。
3. 云函数删除云存储头像时触发异常，虽然本地有 `try/catch`，但后续某一步仍然失败。
4. 云函数已部署，但云端实际运行的版本不是当前本地代码，导致返回结构与前端预期不一致。
5. 某个集合不存在、字段不一致或删除权限异常，导致函数返回失败。

## 下一步

- 先补最小化调试输出，让前端把 `deleteAccount` 云函数的真实返回结果打印出来。
- 如有必要，再根据返回结果定位是集合删除、头像删除还是云函数版本问题。
